# Plan: Fix Audition Conversion & Automate Roster Association

## Objective
Resolve the issue where audition-to-singer conversion fails silently. Automate the association of converted singers with the performance events leading up to their target concert.

## Proposed Changes

### 1. Hardening Profile Creation
- **File**: `src/services/profileService.ts`
- **Action**: Update `createProfile` to automatically generate a random 12-character password if an `email` is provided but a `password` is missing. This prevents the "password required" error during automated conversions.

### 2. Enhancing Conversion Logic
- **File**: `src/services/auditionService.ts`
- **Action**: Update `convertAuditionToSinger` to:
  - Call `profileService.createProfile` (now safe with auto-generated passwords).
  - If the audition had a `performance` association:
    - Fetch all `events` where `parentPerformanceId` matches the audition's performance.
    - Also include the performance event itself.
    - For each event found, create an `eventRosters` entry for the new profile.
  - Mark the audition as `Closed`.

### 3. Improving Admin UI Feedback
- **File**: `src/views/admin/AuditionsView.tsx`
- **Action**: Add a `try...catch` block around the `convertToSinger` call. Use the `dialog` context to show success or error messages to the admin.

### 4. Technical Migration (Optional but good)
- We could add an `email` field to the `profiles` collection to store it even if a user account creation fails, but the current schema relies on the `users` relation. We will stick to the relation for consistency.

## Verification Plan
1. **Manual Test**: Create an audition tied to a performance leading to multiple rehearsals.
2. **Convert**: Click "Convert" in the Auditions dashboard.
3. **Verify Profile**: Ensure the singer profile was created in the Roster.
4. **Verify Roster**: Check the event details for the performance/rehearsals and verify the new singer is listed as "Pending" in the RSVP list.
5. **Verify Error Handling**: Simulate a failure (e.g. duplicate email) and ensure the UI shows a helpful error message.
