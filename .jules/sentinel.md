# Sentinel Journal\n\n## 2024-05-18 - XSS in resolvePreviewContent via unescaped dynamic variables\n**Vulnerability:** dynamic variables in `resolvePreviewContent` like `{eventTitle}`, `{eventDetails}` are not properly HTML escaped before being inserted into the preview HTML which is then rendered via `dangerouslySetInnerHTML`.\n**Learning:** When using native replace for HTML rendering, we must escape the variables before replacement to prevent XSS.\n**Prevention:** Create an `escapeHtml` utility function in `src/lib/communicationUtils.ts` and use it on user-provided dynamic variables in `resolvePreviewContent` before substitution.
